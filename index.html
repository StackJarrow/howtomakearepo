🚀 GitHub + VS Code Workflow

My personal step-by-step guide to creating repos and managing changes

📦 Part 1: Initial Repository Setup
Use this when creating a brand new repository
1
Create GitHub Repository
Go to GitHub website and create a new repository
2
Setup Local Folder
Create folder on computer and open in VS Code
3
Initialize Git
While in that folder (assuming git is setup and logged in through CLI)
git init
4
Create HTML File
Make a file in VS Code called index.html
5
Add Your Code
Put your HTML code in it and add style.css and index.js if that is also part of it
6
Check Status (First Time)
See what files are not added (they will show as untracked)
git status
7
Add Files
Add all files or specific ones
git add .
OR
git add filename.html
8
Verify Status
Check if files were added. If not added they will be in red, if added they will be in green
git status
9
Commit Changes
Save your changes with a description
git commit -m "Your description here"
10
First Push Attempt
Try pushing (this may fail without remote, that's okay)
git push
11
Connect to GitHub
From GitHub webpage for new repo, copy and type the remote add origin command
git remote add origin https://github.com/YOURUSERNAME/REPO-NAME.git
12
Set Upstream (If Fatal Error)
If met with a fatal message, set the upstream branch
git push --set-upstream origin main
💡 This only needs to be done once per repository. After this, regular git push will work.
13
Final Push
Push again - should be met with "Everything is up to date" message
git push
14
Verify on GitHub
Refresh GitHub webpage and see if contents appear
📝 Part 2: Making Changes to Existing Repo
Use this when adding new files or updating existing ones
1
Make Changes
In VS Code, add a new file or make changes and make sure to save them
💡 Tip: Use Ctrl+S (Windows) or Cmd+S (Mac) to save files
2
Check Differences
Type git status to check what is not the same on the GitHub repo - it will appear in red
git status
3
Stage Changes
Add specific file or all saved changes
git add filename.extension
OR add everything:
git add .
4
Verify Staging
Type git status again to check to see if they were added
git status
5
Commit Changes
Add description for your changes
git commit -m "Description of changes"
⚠️ Note: If you do this after adding all new files with git add ., all of their descriptions will be the same as you put in this commit message. If you want separate descriptions for different files, commit them individually.
6
Final Status Check
Type git status again to check (should show nothing to commit or working tree clean)
git status
7
Push to GitHub
Push finally to the repo
git push
